  - 블랙햇으로 SEO를 하면, 도메인 자체가 좋은 평가를 받을 수 없음
- 컨텐츠 SEO : 유저가 원하는 정보를 알기 쉽게 전달하고, 스트레스 없이 페이지를 사용할 수 있도록 해주는 것
- URL 최적화, 중복 컨텐츠 회피 등의 키워드 중요

### Canonical

---

- 한국에는 중복 컨텐츠 회피가 안 되어 있는 사이트가 많다!
- 중복 컨텐츠를 조절하는 것만으로도 테크니컬 SEO에 많은 영향을 끼침
- ex) 쇼핑몰 사이트에서 여러 컬러가 있는 옷을 구매할 때, 어떤 컬러를 선택함에 따라 url이 바뀌지 않으면, 크롤은 여러 컬러 옵션이 있다고 인식하지 못함
  - 이 컬러 옵션의 수만큼 중복 페이지가 있다고 생각하면 됨
- 이런 것들을 회피할 수 있도록 하는 게 Cannonical

### 암호화 통신 대응 | https 도입 | 사이트의 암호화 통신(SSL) 대응

- **GSC(구글 서치 콘솔)**로 http 및 https 여부 확인
  - 웹사이트 뒷면에 있는 데이터를 크롤해서 한번에 모아주는 기능을 함
- mixed contents(예시 중 하나로, 이미지의 URL이 http인 경우) 존재 여부를 확인
- (추천) SEO 분석 도구인 **`Screaming Frog`**로 내부 링크 기술 확인
  - 무료 툴은 아님
- 일반적인 리다이렉트 시 문제 없으나, 만약 302 리다이렉트인 경우, 이전 URL이 계속 잔류할 가능성이 있음

## 네이버 SEO 살펴보기

![](https://efficacious-camel-96c.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F5eda31ee-1593-4ecf-bfad-bae333b90117%2F4ddb2314-6d74-45f7-ad37-7ea1ebcae8ce%2FUntitled.png?table=block&id=051087a6-2e00-4461-9948-8f73bad1a6c3&spaceId=5eda31ee-1593-4ecf-bfad-bae333b90117&width=1420&userId=&cache=v2)

- 서브 도메인
- 도메인/서브 도메인은 구글에서 완전히 다른 사이트로 인식함
- 그래서 도메인 - 서브 도메인이 이어지는 느낌이 아님
- 파라미터로 연결되는 사이트는 동적 URL이기 때문에 정적 URL보다 평가 기준이 낮다
  - 정적/동적 URL을 잘 고려해서 설정하는 게 좋다
- 바뀔 수 없는 URL인 정적 URL은 더 존재감이 있다고 인식된다!

![](https://efficacious-camel-96c.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F5eda31ee-1593-4ecf-bfad-bae333b90117%2Fdd174668-a1e7-4561-8d2e-1649fe5158d1%2FUntitled.png?table=block&id=8b53528d-6b58-450f-8cdb-b851af693344&spaceId=5eda31ee-1593-4ecf-bfad-bae333b90117&width=990&userId=&cache=v2)

- 클릭을 할 때마다 컨텐츠가 바뀌는데 URL이 할당되어 있지 않기 때문에 컨텐츠가 바뀌어도 페이지가 하나만 있다고 인식

→ 페이지네이션을 할 때 URL이 바뀌어야 SEO적으로 좋다!

- 의외로 네이버 바로 밑에 페이지는 별로 없고, 거의 서브 도메인으로 연결됨

- 크롤이 사이트를 이해하는 데 문제가 생기는 경우가 있음 → 어느 게 맞는 건지 혼란이 있을 수 있음

## 무신사 SEO 살펴보기

![](https://efficacious-camel-96c.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F5eda31ee-1593-4ecf-bfad-bae333b90117%2F3284580a-4976-49cb-a231-1ccfc0232576%2FUntitled.png?table=block&id=72dbfbe8-ecf0-4721-b8cd-e7522d1e1aaa&spaceId=5eda31ee-1593-4ecf-bfad-bae333b90117&width=1420&userId=&cache=v2)

- 사람이 볼 때는 14096엔이지만, 크롤러는 ￥14,096 ← 이런 텍스트로 인식
  - 금액이라는 걸 인식하려면 “데이터 구조화” 처리가 필요하다!
- 이런 걸 안 해주면 크롤링이 안 되고, 크롤링이 안 되면 구글 쇼핑탭에서도 검색이 안 됨 → 상품으로 인식이 안 돼서

![](https://efficacious-camel-96c.notion.site/image/https%3A%2F%2Fprod-files-secure.s3.us-west-2.amazonaws.com%2F5eda31ee-1593-4ecf-bfad-bae333b90117%2Fe72df094-e829-4b67-bd71-86670fd21eb2%2FUntitled.png?table=block&id=f13db4d3-8123-47a2-ac90-976c2a5769da&spaceId=5eda31ee-1593-4ecf-bfad-bae333b90117&width=1420&userId=&cache=v2)

- 색상별로 클릭을 할 때 URL은 바뀌는데 Keywords가 없음 → Title과 Description에 색상 관련 키워드가 없다는 말
- 저기 3번째 Keywords는 잘 써봤자 SEO에 아무런 영향을 주지 않음

## 애플 SEO 살펴보기

- 사용자가 이 페이지에서 확인할 수 있는 내용을 Description으로 넣어주는 게 좋음
- 네이버 같은 경우, 저기에 한국적인 표현이 많이 들어 있어서 글로벌적으로는 좋지 않았음

→ 강의자분이 사용하신 리다이렉트 확장 도구

- 의도적으로 글자가 안 보이게 하고 있는 것 → SEO적으로 안 좋다
- 의도적으로 텍스트를 감췄다고 인식되어서 안 하는 게 좋다

- 이미지에 alt 넣는 것 SEO에 도움됨
- h1, h2 같은 태그들도 SEO에 도움됨
